# V9｜權限安全強化與使用者管理版

## 核心修正

1. 正式 Firebase 模式停用 Demo 登入，避免繞過最高管理員審核。
2. 未核准帳號登入後只能看到待審核頁，不能進入系統、不能讀取案件、不能刪除資料。
3. 承辦人只能編輯／刪除／管理自己建立的案件。
4. 系統管理員可管理全部案件，但不能更改最高系統管理員帳號。
5. 最高系統管理員固定為 `fc781117@gmail.com`，前端與 Firestore Rules 皆寫死保護。
6. 系統設定新增完整使用者管理：註冊帳號總數、待審核、已核准使用中、系統管理員、已停用。
7. 使用者可被核准、停用、恢復；最高系統管理員可指定承辦人或系統管理員。
8. Firestore Rules 已同步強化，資料庫層會阻擋未審核帳號與越權刪改。

## 重要提醒

此版本一定要同步更新 Firebase Console 的 Firestore Rules，否則前端更新後仍可能被舊資料庫規則放行。
