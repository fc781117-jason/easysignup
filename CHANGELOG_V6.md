# V6 權限審核與首頁整合版

## 核心更新

1. **新增使用者審核機制**
   - 除最高系統管理員 `fc781117@gmail.com` 外，所有 Google 登入帳號第一次登入後會進入「待審核」狀態。
   - 待審核者無法進入系統首頁、案件、報名與設定功能。
   - 最高系統管理員可在「系統設定 → 使用者審核」核准或停用使用者。

2. **強化案件權限**
   - 承辦人只能管理、編輯、刪除自己建立的案件。
   - 非本人建立的案件可查看與報名，但不能進入單一案件管理。
   - 最高系統管理員可管理、編輯、刪除所有案件。

3. **首頁整合**
   - 移除獨立「儀表板」選單。
   - 將原案件管理整合為「首頁」。
   - 首頁上方僅保留兩個小型統計：開放中案件、已截止案件。
   - 下方直接顯示案件清單，可進行報名填寫、重新編輯或案件管理。

4. **文案調整**
   - 案件清單操作按鈕由「填寫」改為「報名填寫」，降低使用者混淆。

5. **Firestore Rules 更新**
   - 新增 `isApproved()` 判斷。
   - 未核准使用者不得讀寫案件、報名與操作紀錄。
   - 案件更新與刪除限定最高系統管理員或案件建立者。
